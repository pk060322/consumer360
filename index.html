rfm_engine.py — core segmentation logic 
def assign_segments(rfm_scored):
    def get_segment(row):
        r, f, m = row['r_score'], row['f_score'], row['m_score']
        total   = row['rfm_total']

        if   r >= 4 and f >= 4 and m >= 4:  return 'Champions'
        elif r >= 3 and f >= 3 and m >= 3:  return 'Loyal Customers'
        elif r >= 3 and total >= 9:          return 'Potential Loyalists'
        elif r >= 4 and f == 1:              return 'New Customers'
        elif r <= 2 and (f >= 2 or m >= 3):  return 'At Risk'
        elif r == 1 and f == 1 and m <= 2:  return 'Lost'
        else:                                   return 'Needs Attention'

    rfm_scored['segment'] = rfm_scored.apply(get_segment, axis=1)
    return rfm_scored

Phase 04 — RFM Segmentation

5 Customer Segments

Every customer is scored 1–5 on Recency, Frequency, and Monetary value — then placed into an actionable segment.

🏆

Champions

R≥4, F≥4, M≥4
Reward & get referrals

💚

Loyal Customers

R≥3, F≥3, M≥3
Upsell & cross-sell

🌱

Potential Loyalists

R≥3, total≥9
Offer loyalty programme

⚠️

At Risk

R≤2, was active
Win-back campaign

💤

Lost

R=1, F=1, M≤2
Reactivation or write off

RFM Score example: A customer with R=5, F=4, M=3 receives an rfm_score = "543" and an rfm_total = 12 — placing them in the Loyal Customers segment. The score string lets you instantly filter in SQL: WHERE rfm_score LIKE '5%' returns all highly-recent customers regardless of frequency or monetary score.

Deployment Guide

Upload to GitHub & Netlify

Follow these steps to publish your project and share it with recruiters and the world.

Part A — GitHub Repository

1

Create a GitHub account

Go to github.com → Sign up (free). Choose a professional username — recruiters will see it.

2

Create a new repository

Click + → New repository. Name it consumer360. Set to Public. Do NOT initialise with README (we already have one).

3

Install Git & push

Download the project ZIP, unzip it, then run:

cd consumer360 git init git add . git commit -m "Initial commit: Consumer360" git remote add origin https://github.com/pk060322/consumer360.git git push -u origin main
4

Add repository topics

On GitHub → Settings → Topics. Add: data-analytics mysql python rfm power-bi

Part B — Netlify Website

1

Create Netlify account

Go to netlify.com → Sign up with GitHub. It's free.

2

Deploy from GitHub

Netlify Dashboard → Add new site → Import an existing project → Connect to GitHub → Select consumer360 repo.

3

Configure build settings

Netlify will auto-detect the index.html. No build command needed. Publish directory: leave blank (root). Click Deploy site.

4

Get your live URL

Netlify gives you a URL like consumer360-abc.netlify.app. Go to Domain settings → rename it to something clean like consumer360.netlify.app.

5

Update README & GitHub

Paste your Netlify URL into your GitHub README and LinkedIn profile. Future pushes to GitHub auto-deploy to Netlify.

⚡ Faster option — Netlify Drag & Drop

Go to app.netlify.com/drop and drag the entire consumer360 folder directly onto the page. Your site goes live in 30 seconds — no account needed to test. Then claim the site with a free account to keep the URL permanent.
